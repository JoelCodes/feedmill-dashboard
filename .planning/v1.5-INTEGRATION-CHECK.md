## New Finding: INT-07 — BLOCKER

**Connection:** `src/components/CustomerOrdersTab.tsx:159` → `/demo/orders` route

**Status:** BROKEN

**Evidence:**
```
src/components/CustomerOrdersTab.tsx:159
  href={`/orders?selected=${order.id}`}
```

**Render chain (active):**
```
/demo/customers/[id]/page.tsx:41
  → <CustomerDetailTabs events={events} orders={orders} />
      (src/components/CustomerDetailTabs.tsx:49)
  → <CustomerOrdersTab orders={orders} />
      (src/components/CustomerOrdersTab.tsx:159)
  → <Link href={`/orders?selected=${order.id}`}>   ← BROKEN: /orders is 404
```

**Impact:** A demo user on `/demo/customers/[id]` who clicks the "Orders" tab and then clicks any individual order row navigates to a 404. This is the same user-visible break class as INT-01/FLOW-01, on the same page, through a sibling component that Phase 29's scope did not enumerate.

**Affected requirement:** ROUTE-01 — "Existing pages moved to /demo/* subdirectory." The page moved; this consumer was not updated.

**Fix:** Change `href` at `CustomerOrdersTab.tsx:159` from `` `/orders?selected=${order.id}` `` to `` `/demo/orders?selected=${order.id}` ``. Add a Jest component test asserting the demo-prefixed href shape (parallel to D-06 for Timeline).

The audit's grep evidence (`grep -n 'selected' src/components/ui/Timeline.tsx`) and scope description named only Timeline.tsx. CustomerOrdersTab.tsx is a peer client component in `src/components/` (not `src/components/ui/`) and was not enumerated in Phase 29's stale-href audit. The class of bug is identical.
