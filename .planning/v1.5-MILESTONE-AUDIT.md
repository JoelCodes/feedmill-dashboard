**INT-07 — BLOCKER**

- **From:** `src/components/CustomerOrdersTab.tsx:159`
- **To:** `/orders?selected=<orderId>` (deleted route, 404)
- **Expected:** `/demo/orders?selected=<orderId>`
- **Affected requirement:** ROUTE-01
- **Render chain (active):**
  ```
  /demo/customers/[id]/page.tsx:41
    → <CustomerDetailTabs orders={orders} />
        (src/components/CustomerDetailTabs.tsx:49)
    → <CustomerOrdersTab orders={orders} />
        (src/components/CustomerOrdersTab.tsx:159)
    → <Link href={`/orders?selected=${order.id}`}>  ← BROKEN: /orders is 404
  ```
- **Why missed by Phase 29:** Phase 29's INT-01 scope and grep evidence enumerated only `src/components/ui/Timeline.tsx`. `CustomerOrdersTab.tsx` lives in `src/components/` (not the `ui/` subdirectory) and was not searched for the same class of stale-href bug.
- **Fix:** Single-line href update (`/orders` → `/demo/orders`) plus a Jest assertion mirroring D-06 from Phase 29.
