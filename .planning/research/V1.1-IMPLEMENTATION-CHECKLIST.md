### 1.2 Filter Logic with Correct Count Calculations

- [ ] **Implement filteredOrders using useMemo** (NOT useEffect)
  ```typescript
  const filteredOrders = useMemo(() => {
    let result = orders;

    if (activeStates.size > 0) {
      result = result.filter(o => activeStates.has(o.state));
    }

    if (activeMills.size > 0) {
      result = result.filter(o => activeMills.has(o.millLine));
    }

    if (searchTerm) {
      const lower = searchTerm.toLowerCase();
      result = result.filter(o =>
        o.customer.toLowerCase().includes(lower) ||
        o.product.toLowerCase().includes(lower)
      );
    }

    return result;
  }, [orders, activeStates, activeMills, searchTerm]);
  ```

- [ ] **Implement stateCounts with self-referential exclusion**
  ```typescript
  const stateCounts = useMemo(() => {
    let toCount = orders;

    // Include mill and search filters
    if (activeMills.size > 0) {
      toCount = toCount.filter(o => activeMills.has(o.millLine));
    }

    if (searchTerm) {
      const lower = searchTerm.toLowerCase();
      toCount = toCount.filter(o =>
        o.customer.toLowerCase().includes(lower) ||
        o.product.toLowerCase().includes(lower)
      );
    }

    // DON'T filter by activeStates (self-referential)

    // Count by state
    const counts: Record<ProductionState, number> = {
      Completed: 0,
      Mixing: 0,
      Blocked: 0,
      Pending: 0,
    };

    toCount.forEach(order => {
      counts[order.state]++;
    });

    return counts;
  }, [orders, activeMills, searchTerm]);
  ```

- [ ] **Implement millCounts similarly**

  const millCounts = useMemo(() => {
    let toCount = orders;

    // Include state and search filters

  ```typescript
  const VALID_STATES: ProductionState[] = ['Completed', 'Mixing', 'Blocked', 'Pending'];
  const VALID_MILLS: MillLine[] = ['Premix', 'Excel', 'CGM'];

  function validateProductionOrder(data: unknown): ProductionOrder {
    if (typeof data !== 'object' || !data) throw new Error('Invalid order');
    const order = data as Record<string, unknown>;

    if (!VALID_STATES.includes(order.state as ProductionState)) {
      throw new Error(`Invalid state: ${order.state}`);
    }

    if (!VALID_MILLS.includes(order.millLine as MillLine)) {
      throw new Error(`Invalid mill line: ${order.millLine}`);
    }

    if (typeof order.customer !== 'string' || !order.customer) {
      throw new Error('Invalid customer');
    }

    return order as ProductionOrder;
  }
  ```

## Troubleshooting During Implementation

### Issue: Filter counts don't match filtered results

**Prevention done in Phase 1:**
- Verify filter matrix is correct (which filters exclude self)
- Check that stateCounts excludes state filter
- Log both values to console and compare

**Fix:**
1. Review count calculation matrix
2. Verify useEffect dependencies correct (or use useMemo)
3. Test with specific example: select one status, verify counts

### Issue: Filters not working with service data

**Prevention done in Phase 2:**
- Runtime validation implemented
- Mock data factory ensures correct structure
- Integration test validates filter + service

**Fix:**
1. Check that service data matches ProductionOrder interface
2. Verify field names exact (case-sensitive)
3. Run validation function directly on service data

### Issue: Design doesn't match (colors/spacing off)

**Prevention done in Phase 3:**
- Design token extraction spreadsheet created
- Overlay tool used to verify pixel-perfect

**Fix:**
1. Compare actual colors to extracted values
2. Check if using correct Tailwind class (bg-primary vs bg-blue-500)
3. Use eyedropper to verify actual rendered color
4. Check if custom shadow/spacing needs adjustment in config

### Issue: "Cannot read property X of undefined"

**Root cause:** Service data missing expected field
**Fix:**
1. Check runtime validation (is it enabled?)
2. Verify mock data factory includes all fields
3. Add console.log to see actual data shape
4. Check DTO mapper if using real API
