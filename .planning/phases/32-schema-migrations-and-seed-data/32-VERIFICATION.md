### 2. Seed Runtime Inline Fix (Execution-Time Discovery â€” Planning-Process Note)

Plan 05 produced `src/db/seed-data.json` with snake_case column keys (e.g., `order_number`, `weight_lbs`). Plan 06's initial `seed.ts` passed the raw JSON to `db.insert(productionOrders).values(seedData as any)`, but Drizzle's ORM layer expects camelCase property names matching the TypeScript schema (e.g., `orderNumber`, `weightLbs`). The mismatch caused the seed to fail on the first live run. Commit `4655b30` fixed this by adding a per-row mapper in `seed.ts` that translates the snake_case JSON shape to camelCase Drizzle properties before insert. The 33-row seed was then confirmed idempotent across two runs.

The fix is now part of the committed `seed.ts` and the snake_caseâ†’camelCase mapper is clearly typed with an inline `SnakeRow` type alias. The seed runtime is correct and complete in the codebase.

**Severity:** Planning-process learning â€” snake_case JSON shape vs. camelCase Drizzle property contract should be verified at plan time for insert operations. No ongoing risk.

The full Jest suite reports 1 failed test suite: `src/app/settings/__tests__/page.test.tsx` (14 failing tests, all under "MIG-04 Design System Migration").
