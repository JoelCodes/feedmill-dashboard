<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser URL → useQueryStates | `?status=`, `?q=`, `?order=` are tamper-able by the user. Same surface as plan 34-01 T-34-01-02 (parsers drop unknown literals). |
| Client filter → render | filterOrders is pure; cannot leak data outside the orders prop (which the page RSC already gated by auth + role). |
| Browser timer → router.refresh | Same surface as plan 34-02 T-34-02-*. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-05-01 | T — Tampering | `?status=BadValue,Pending` URL | mitigate | parseAsStringLiteral drops unknown literals silently (plan 34-01 Test 2). Dashboard receives a clean ProductionState[] array — no XSS or injection. |
| T-34-05-02 | T — Tampering | `?q=<script>alert(1)</script>` URL | mitigate | `q` is used only as a substring needle in `filterOrders` (toLowerCase + .includes); React escapes it when rendering inside the controlled search input (no `dangerouslySetInnerHTML`). The displayed value in the input is the controlled `searchInput` state which React renders as text. No XSS surface. |
| T-34-05-03 | I — Information Disclosure | `?q=` appears in browser history | accept | Search terms are operator-internal; no PII. Documented in CONTEXT.md T-34-01-04. |
| T-34-05-04 | D — Denial of Service | Search input spam | mitigate | 150ms debounce + nuqs internal batching means a fast typist generates ≤ 1 URL update per ~150ms. No exploitable spam vector. |
| T-34-05-05 | E — Elevation of Privilege | canEdit prop unused in plan 05 | accept | Plan 06 will consume canEdit when rendering the drawer's transition buttons. In plan 05, hiding/showing the dashboard does NOT depend on canEdit (D-25 — read-only viewers see the same board minus drawer transition controls). |
| T-34-05-06 | I — Information Disclosure | Click on a card sets `?order=<id>` (bookmarkable URL) | accept | Order IDs are opaque internal identifiers; bookmarking is intentional (D-06). Auth gate on `/` still enforces visibility. |
