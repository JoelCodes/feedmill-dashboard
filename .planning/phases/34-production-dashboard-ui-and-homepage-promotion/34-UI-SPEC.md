# Phase 34 ‚Äî UI Design Contract

> Visual and interaction contract for the production dashboard UI and homepage promotion.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Tailwind v4 + CSS custom properties) |
| Preset | not applicable |
| Component library | custom ‚Äî `src/components/ui/` (Button, Card, FilterPill, StatusBadge, Textarea, Input, Select, Timeline) |
| Icon library | lucide-react (already installed; used throughout project) |
| Font | Helvetica, Arial, sans-serif (defined in `body` in `globals.css`) |

No shadcn. All design tokens live in `src/app/globals.css` as CSS custom properties. Tailwind v4 `@theme inline` block maps tokens to utility classes. The executor MUST use existing token names ‚Äî no hardcoded hex values.

## Spacing Scale

All values from `globals.css` `--space-*` tokens. Declared values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-1`) | Icon gaps, dot spacing, inline padding |
| sm | 8px (`--space-2`) | Compact element spacing, pill gap, card inner row spacing |
| md | 16px (`--space-4`) | Default element spacing, card padding horizontal |
| lg | 24px (`--space-5`) | Section padding, column gap, card outer gap |
| xl | 32px (`--space-6`) | Layout gaps between columns |
| 2xl | 48px (`--space-8`) | Major section breaks |
| 3xl | 64px (`--space-10`) | Page-level spacing |

Exceptions:
- Filter pill height: 32px (h-8) ‚Äî fixed from `FilterPill` component
- Nav icon container: 30px (--nav-icon-size) ‚Äî fixed from sidebar pattern
- Column skeleton cards: match card padding-lg (21px, --card-padding-lg)
- Drawer width: 480px ‚Äî wide enough for all order fields + event timeline in a single-column layout
- Blocked alert band: 12px top/bottom padding (--space-3), 16px left/right (--space-4)
- Touch targets for transition buttons: minimum 40px height (h-10, Button size `md`)

## Typography

All sizes from `globals.css` `--fs-*` tokens. Body font is Helvetica/Arial from `globals.css`.

| Role | Size | Weight | Line Height | Token / Usage |
|------|------|--------|-------------|---------------|
| Label (card meta) | 11px (`--fs-11`) | 600 (semibold) | 1.4 | Order number, status label text, pill text, section labels |
| Body (card body) | 15px (`--fs-15`) | 700 (bold) | 1.4 | Customer name on card, primary card title |
| Body (standard) | 14px (text-sm) | 400 (regular) | 1.5 | Drawer field values, import table rows, helper text |
| Heading | 24px (text-2xl) | 700 (bold) | 1.2 | Mill column headers (Premix / Excel / CGM) |

Weights used: **400 (regular)** and **700 (bold)**. Semibold (600) is used only for label-tier text (11px labels, pill text, breadcrumbs).

Note: The existing `--fs-10` (10px) is used for count bubbles inside pills and section labels (e.g., "PRODUCTION"). This is a deliberate exception for metadata badges ‚Äî not body text.

## Surface-by-Surface Contracts

### 1. Board Header Strip (top of `ProductionDashboard`)

Layout: single horizontal row, full board width.

Left side:
- Four `FilterPill` components (Completed / Mixing / Blocked / Pending) with production state color configs. Gap: 10px (gap-2.5).
- Search input: same visual style as existing `Header` search field ‚Äî white bg-card, shadow-sm, Search icon (lucide), 12px placeholder text in `--text-secondary`.

Right side (flex-end):
- "Import Orders" button ‚Äî `Button` variant `secondary`, size `sm`, icon: `Upload` (lucide). Links to `/import`.
- `LastUpdatedChip` ‚Äî text format: "Updated Xs ago", 11px, `--text-secondary`. Refresh icon button (`RotateCcw` lucide) next to it; on click: `router.refresh()` + spinner overlay on the icon.

Search debounce: 150ms. URL key: `?q=`. Trimmed and lowercased before apply; URL preserves original case.

### 2. Blocked Alert Band

Visibility: hidden (component returns `null`) when `blockedOrders.length === 0`.
When visible: sticky band pinned between the header strip and the 3-column board. Scrolls with page (not fixed), sticky within the board container viewport.
Background: `--error-light` tint (`bg-error-light`). Left accent: 4px solid `--status-blocked-border`.
Content: Each blocked order rendered as a clickable chip: `"BLOCKED: [Document Number] ([MillLine]) ‚Äî [reason excerpt, max 40 chars]"`. Font: 12px (text-xs), `--error-dark`. Click: sets `?order=<id>`.
Padding: 12px top/bottom, 16px left/right.
Layout: `flex flex-wrap gap-2` for chips.

### 3. Mill Columns (three-column board)

Column container: `flex gap-6` (24px gap) matching demo.
Each column: `flex flex-1 flex-col gap-5`.
Column header:
- Mill line name: 24px bold, `--primary` color (matches `MillProductionUI.tsx` pattern exactly)
- Sub-label: "X / Y lbs" format, 14px, semibold, `--text-muted`

State sections within column: label (state name) in state header color at 20px bold, weight total beside it at 14px medium `--text-muted`. State sections in order: Completed ‚Üí Mixing ‚Üí Blocked ‚Üí Pending (same as demo). Sections with zero orders return null (no empty section headers).

Suspense boundary per column: `<Suspense fallback={<ColumnSkeleton />}>`.

### 4. Production Card

Shape: same as demo `ProductionCard`. Left border stripe: 4px solid, color from status token (`STATE_COLORS[order.state].border`). Rounded: `rounded-r-xl`. Background: `--bg-card`. Shadow: `--shadow-card`.
Content (top-to-bottom):
1. Order number ‚Äî 11px, semibold, `--text-muted` (label tier)
2. Customer name ‚Äî 15px (--fs-15), bold, `--text-primary` (title tier)
3. Weight + product ‚Äî 14px (text-sm), medium, `--text-medium`; weight formatted as `Xk` if ‚â•1000 lbs, else comma-locale
4. Delivery: [delivery_time] ‚Äî 12px (text-xs), medium, `--text-muted`

Special indicators on the card:
- **Next-up indicator** (PROD-07): topmost Pending order per column. Render a small teal pill above the order number: `"Next Up"` in 10px bold white text, `--primary` background, 4px border-radius, padding 1px 6px. Only the first Pending order per column gets this; detected by sorted position.
- **In-progress badge** (PROD-08): every Mixing order. Render an animated dot beside the order number: 8px circle, `--status-mixing-header` color, `animate-pulse`. Label: "Mixing" text is already visible from the state section heading; no additional badge text needed on the card. The animated dot suffices.

Interactive: `cursor-pointer`, `hover:opacity-95`, `active:scale-[0.98]` (matches existing `Card` interactive pattern). On click: set `?order=<id>` via nuqs.

### 5. Order Details Drawer

Width: 480px. Fixed to the right edge of the viewport, overlays the board.
Backdrop: semi-transparent overlay behind the drawer using `bg-black/30` covering the board area. Click backdrop ‚Üí close drawer (clear `?order=`).
Close gestures: drawer-X button (top-right), outside-click (backdrop click), ESC key.
Animation: slide-in from right, `translate-x-full` to `translate-x-0`, 200ms ease-out.

Header area:
- Order number: 11px semibold `--text-secondary`
- Customer name: 20px bold `--text-primary`
- State badge: `StatusBadge` component (needs production state config)
- X close button: top-right, `ghost` Button size `sm`, `X` lucide icon

Fields section (label + value pairs, 2-column grid or stacked):
- Document Number, Customer, Product, Weight (lbs), Delivery Date, Mill Line, Formula Type (texture_type), Blocker Note (if state is Blocked)
- Labels: 11px semibold `--text-secondary`
- Values: 14px regular `--text-primary`
- Weight: `parseFloat(weight_lbs).toLocaleString()` + " lbs"
- Clerk user ID rendered directly as `changed_by` in event timeline (no display-name lookup in v2.0)

Divider: `h-px bg-[var(--divider)]` between sections.

Timeline section (order_events):
- Heading: "Transition History", 14px semibold `--text-primary`
- Reuse existing `Timeline` component from `src/components/ui/Timeline.tsx` if compatible; otherwise render chronologically as a `flex flex-col gap-3` list.
- Each event: dot (8px, status color for `to_state`), `[from_state] ‚Üí [to_state]` in 12px medium, date+time in 11px `--text-muted`, note (if present) in 12px italic `--text-secondary`

Transition buttons (visible only when `canEdit === true`):
- Pending state: single `Button` primary "Start Mixing"
- Mixing state: `Button` primary "Complete Order" + `Button` destructive "Block Order"
- Blocked state: split pattern ‚Äî `Button` primary "Resume to Mixing" + `Button` secondary "Resume to Pending"
- Completed state: no action buttons
- Buttons grouped at bottom of drawer in `flex gap-3`, full-width each (w-full)

**Conflict error banner** (TRANS-06): When action returns `{ code: 'conflict' }`, show inline red banner inside drawer above buttons. Background `--error-light`, left border 4px `--error`, text `--error-dark` 13px. Locked message: `"Order was modified by another user. Please refresh."` Auto-calls `router.refresh()`.

**Unauthorized banner**: When action returns `{ code: 'unauthorized' }`, redirect to `/sign-in` (matches Phase 33 D-02).

**`?order=` stale ID state**: Draw renders "Order not found" in 14px `--text-secondary`, centered, with a "Close" ghost button. Does NOT throw.

Suspense boundary: `<Suspense fallback={<DrawerSkeleton />}>` wraps the drawer server component.

### 6. Block Reason Modal

Trigger: "Block Order" button in drawer.
Primitive: `@radix-ui/react-dialog`. Stacks on top of drawer (z-index above drawer + backdrop).
Backdrop: `bg-black/50` overlaying the drawer.
Modal surface: `bg-[var(--bg-card)]` rounded `--radius-lg`, max-width 480px, centered.
Content:
- Title: "Block Order" ‚Äî 16px bold `--text-primary`
- Textarea label: "Reason (required)" ‚Äî 14px medium `--text-primary`
- Textarea: reuse existing `Textarea` component, `minRows=3`, placeholder "Describe the issue..."
- Empty textarea: Confirm button disabled (`disabled:opacity-50`) client-side
- Buttons: `Button` primary "Confirm Block" + `Button` secondary "Cancel". Cancel closes modal, returns to drawer.
- On server validation error: show inline error beneath textarea using `Textarea` `error` prop.
ESC / click outside backdrop closes modal (Radix Dialog default behavior). This must NOT propagate to close the underlying drawer ‚Äî Radix Dialog captures events within its Portal.

### 7. Loading Skeletons

**ColumnSkeleton**: 3 animated card placeholders. Each card: `h-20 w-full animate-pulse rounded-r-xl bg-[var(--divider)]`. Gap between cards: 12px (gap-3). Column header skeleton: `h-6 w-24 animate-pulse rounded bg-[var(--divider)]` + `h-4 w-16`.

**DrawerSkeleton**: Matches drawer shape (480px width). Header area: two lines. Fields: 6 label+value rows, `h-3 w-24` and `h-4 w-full` animated. Timeline: 3 dot+text rows. Matches pattern of existing `DetailsSkeleton.tsx`.

### 8. Import Page (`/import`)

Layout: Full page wrapped in `DashboardLayout`. Two-section page: top = import flow, bottom = history table.

**Drop zone entry state**:
- Dashed border `--divider`, rounded `--radius-lg`, background `--bg-card`, min-height 200px, centered content.
- Icon: `Upload` (lucide), 32px, `--text-secondary`.
- Primary text: "Drop your Excel file here" ‚Äî 16px bold `--text-primary`.
- Secondary text: "or click to browse ‚Äî .xlsx only, max 2 MB" ‚Äî 13px `--text-secondary`.
- Drag-over state: border changes to `--primary` (2px solid), background `--primary` at 5% opacity.
- File-size error (>2MB): inline error below drop zone, 13px `--error`, icon `AlertCircle` (lucide).

**Preview state** (after `previewImportAction` returns):
- Summary bar: "X rows ¬∑ Y lbs total ¬∑ Z duplicates detected" ‚Äî 14px medium `--text-primary`. Background `--pending-light`, padding 8px 16px, rounded `--radius-md`.
- Preview table: full-width, standard table styling (header row with `--text-secondary` 11px semibold, data rows 13px `--text-primary`). Columns: Row #, Document Number, Customer, Product, Weight, Status (new / duplicate). Duplicate rows: background `--warning-light` tint.
- Per-row Skip/Overwrite radio (duplicate rows only): default Skip. Radio labels 12px `--text-medium`.
- Row-level validation errors: text in `--error` 12px below the offending row.
- "Commit Import" button: `Button` primary, bottom-right. Disabled when zero valid rows.
- "Cancel" button: `Button` secondary, beside Commit. Returns to drop zone state (clears preview).

**Import history table** (below a `--divider` horizontal rule):
- Section heading: "Recent Imports" ‚Äî 16px bold `--text-primary`.
- Table columns: File Name, Rows, Imported By (Clerk user ID), Date. Widths: auto.
- Date format: `MMM D, YYYY h:mm a` (e.g., "May 14, 2026 2:34 pm").
- Empty state: "No imports yet" centered 14px `--text-secondary`.
- Row count: 10 most recent batches.

**"Import Orders" button** on board header strip links to `/import`. Browser back returns to `/` with URL state preserved.
