<objective>
Close two coupled gaps from UAT.md T9:

- **T9a** (major): `ImportFlow.handleCommit` does not call `router.refresh()` after a successful `commitImportAction`. The history table stays stale until manual reload. Server-side `revalidateTag('import-batches', 'max')` is correctly wired (`src/actions/import.ts` lines 793 + 808); the gap is purely client-side.
- **T9b** (blocker): `ImportFlow` is a `'use client'` component. The `batches: ImportBatch[]` prop crosses the RSC→client boundary, turning `Date` into ISO strings. `ImportHistoryTable.tsx:27` then calls `Intl.DateTimeFormat.format(string)` which throws `RangeError: Invalid time value`.

**Why one plan:** The fixes are tightly coupled. T9a's `router.refresh()` causes the page RSC to re-fetch and pass a fresh `batches` array to `ImportFlow`. Without T9b's hydration in place, that fresh array still arrives as strings and the post-commit refresh immediately surfaces the RangeError to the operator. Sequencing T9b first, then T9a, in one plan is the safe order.

Per the UAT routing decision (table, 2026-05-14): "T9b → Hydrate `importedAt` in ImportFlow before passing down + regression test" (Option B from the debug session). NOT Option A (`new Date()` inside `formatBatchDate`) — that masks the type lie. NOT Option C (move table render server-side) — too large for gap closure.

Purpose: Make the operator-visible XLSX import flow actually complete end-to-end. After this plan, an operator can drop a valid file, click Commit, see "Import complete!", and immediately see the new batch at the top of the history table — without a hard refresh and without a runtime error.

Output:
- `ImportFlow.tsx` hydrates `batches[].importedAt` from `string|Date` to `Date` (via `new Date(...)`) at the prop boundary BEFORE passing to `ImportHistoryTable`.
- `ImportFlow.tsx` imports `useRouter` from `next/navigation` and calls `router.refresh()` inside `handleCommit` immediately after `commitImportAction` returns `{ ok: true }` (mirroring the pattern at `TransitionButtons.tsx:69`).
- `ImportHistoryTable.test.tsx` gains a regression test that passes a string-typed `importedAt` and asserts the component renders without throwing (proves the architecture is no longer load-bearing on TS's optimistic `Date` type).
- `ImportFlow.test.tsx` gains a test that mocks `useRouter`, invokes the commit flow with a successful `commitImportAction` result, and asserts `mockRefresh` was called.
