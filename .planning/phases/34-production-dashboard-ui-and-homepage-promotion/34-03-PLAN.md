## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RSC → Drizzle/Neon | `getImportBatches` is `'server-only'`; cannot be called from a client component. Tag string matches the action-side patch (D-21). |
| Client → URL state | BlockedAlertBand click writes `?order=<id>` via nuqs — same tampering surface as plan 34-01 (T-34-01-02). |
| Browser timer → router.refresh | Same surface as plan 34-02 polling hook. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-03-01 | S — Spoofing | getImportBatches consumer | accept | `'server-only'` import + Drizzle parameterised queries; cannot be spoofed from client. RSC consumer is the page in plan 34-07 which enforces `auth()` redirect. |
| T-34-03-02 | T — Tampering | `?order=<id>` set by BlockedAlertBand chip click | mitigate | `parseAsString` accepts any string; `getOrderById(id)` (called from plan 07 page RSC) uses parameterised Drizzle query — no SQL injection surface. Stale ID renders empty state per Pitfall 7 / UI-SPEC §5. |
| T-34-03-03 | R — Repudiation | getImportBatches has no audit row | accept | Read-only operation; auditing reads has no value for this milestone. Mutation auditing is still satisfied by `import_batches` rows (Phase 33). |
| T-34-03-04 | I — Information Disclosure | BlockedAlertBand displays `orderNumber` + `millLine` | accept | Both are operator-internal (Document Number + mill line). No PII. `BLOCKED:` prefix is operational labelling, not customer-visible (page is auth-only). |
| T-34-03-05 | I — Information Disclosure | LastUpdatedChip relative-time exposes session activity timing | accept | Relative-time of last data refresh is benign metadata. No correlation path to user identity beyond what is already exposed by the dashboard render itself. |
| T-34-03-06 | D — Denial of Service | LastUpdatedChip 5s tick + manual refresh spam | mitigate | 5-second `setInterval` is single-threaded and cleaned up on unmount. Manual refresh button disables itself during `isRefreshing` state (Test 6) — prevents click-spam triggering N parallel refreshes. |
| T-34-03-07 | E — Elevation of Privilege | None of these components surface mutating actions | accept | All four are read/render-only. No `requireRole` needed. |
| T-34-03-08 | I — Information Disclosure | Cached query content visible to operators | accept | Same trust model as `getProductionOrders` (Phase 33). `/import` page in plan 07 enforces `auth()` redirect. |
</threat_model>
