    Function: sortCustomersByRecentActivity(customers: CustomerWithStats[]): CustomerWithStats[]

    Cases:
    - Input: [customer with delivery 2026-03-15, customer with delivery 2026-03-10]
      Output: [customer with 2026-03-15 first, then 2026-03-10] (most recent first)

    - Input: [customer with no orders, customer with delivery 2026-03-10]
      Output: [customer with delivery first, customer with no orders last]

    - Input: [customer A no orders, customer B no orders]
      Output: Stable order preserved (both at end, order unchanged)

    - Input: []
      Output: []

    - Input: [single customer]
      Output: [same single customer]

    Edge cases:
    - Customers with same delivery date: stable sort (preserve original order)
    - All customers have no orders: return in original order
  </behavior>
  <implementation>
    1. Import mockOrders from @/services/mockData to access delivery dates
    2. Create helper getMostRecentDeliveryDate(customerId: string): Date | null
       - Filter mockOrders by customerId
       - If no orders, return null
       - Return max deliveryDate from customer's orders
    3. Create sortCustomersByRecentActivity function:
       - Use spread operator to avoid mutating input ([...customers].sort())
       - Sort comparator:
         - Get dates for both customers
         - If a has no date (null), return 1 (move to end)
         - If b has no date (null), return -1 (a stays before)
         - Otherwise compare: bDate.getTime() - aDate.getTime() (descending)
  </implementation>
