## Data Display Patterns

### Customer Row
- **Left column (fill):** Customer name (12px, weight 700, color #2d3748)
- **Right column (120px fixed):** Status indicators (horizontal, 8px gap)

### Status Indicators Priority (left to right)
1. Package icon (#4fd1c5) — Always shown if customer has active orders
2. Red dot (8px circle, #e53e3e) — Shown if customer has orders with changes
3. Alert triangle — Shown if customer has bin alerts
   - Yellow (#f59e0b) for low-level alerts
   - Red (#e53e3e) for critical alerts

**Empty indicators column:** Show nothing (not placeholder dashes)

**Source:** customers.pen row structure and Phase 11 data types

## Accessibility Notes

- Search input must have proper label (visually hidden or aria-label)
- Customer rows must be keyboard navigable (Enter/Space to activate)
- Status icons must have aria-label for screen readers
- Empty state must be announced to screen readers
- Table must use semantic HTML or ARIA table roles

**Deferred to implementation:**
- Focus management after search
- Keyboard navigation between rows
