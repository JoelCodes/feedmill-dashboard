  <action>
The Card component has three jsx-a11y violations when onClick is provided:
1. click-events-have-key-events - needs keyboard handler
2. no-static-element-interactions - div with onClick needs role
3. no-noninteractive-tabindex - tabIndex on div with role="button" triggers this

The fix is to add keyboard support and ensure the role/tabIndex combination is correct.

Update the Card component (around line 29):

```typescript
function Card({ variant, className, children, onClick, ...props }: CardProps) {
  const isClickable = !!onClick;

  const handleKeyDown = (e: React.KeyboardEvent<HTMLDivElement>) => {
    if (onClick && (e.key === 'Enter' || e.key === ' ')) {
      e.preventDefault();
      onClick(e as unknown as React.MouseEvent<HTMLDivElement>);
    }
  };

  return (
    <div
      className={cn(
        cardVariants({ variant }),
        isClickable &&
          "cursor-pointer hover:opacity-95 transition-opacity active:scale-[0.98]",
        className
      )}
      role={isClickable ? "button" : undefined}
      tabIndex={isClickable ? 0 : undefined}
      onClick={onClick}
      onKeyDown={isClickable ? handleKeyDown : undefined}
      {...props}
    >
      {children}
    </div>
  );
}
```

This satisfies:
- click-events-have-key-events: onKeyDown handler added
- no-static-element-interactions: role="button" makes it interactive
- no-noninteractive-tabindex: with role="button", tabIndex=0 is valid on an interactive element
