<objective>
Ship `commitImportAction` — the write half of the XLSX import flow. It is the second action in `src/actions/import.ts`; it re-parses the uploaded file per D-05 (stateless server — abandoned previews leave no server-side state), re-validates rows, applies the operator's `ImportDecisions` (per-row skip/overwrite), inserts or updates production_orders one row at a time (partial-import per D-08 + IMPORT-04), writes audit events for both initial inserts AND overwrites, and writes ONE `import_batches` row at the end (D-07).

Purpose: Closes IMPORT-04 (partial-import semantics with row-level error reporting), IMPORT-05 (duplicate detection + skip/overwrite per row), and IMPORT-06 (`import_batches` row on success). Closes ROADMAP SC#5 (operator can upload Book1.xlsx-format file, preview shows row count/total weight/duplicates, confirmed imports appear in import_batches log, files >2MB rejected).

This plan adds to the same `src/actions/import.ts` file that plan 33-05 created. The shared helpers (`parseAndValidate`, `detectIntraFileDuplicates`, `detectDbDuplicates`, `dateToIsoString`, and the `xlsxSchema` const) are re-used. The commit-only additions are: the `commitImportAction` export, `ImportDecisions` and `CommitResult` type exports, the per-row insert/update loop, the overwrite event row pattern, the `import_batches` insert, and the `revalidateTag` call.

Output: Extended `src/actions/import.ts`. One new test file. The total Phase 33 surface (queries, transitions, import preview + commit, config, deps) is complete after this plan.

export type ImportDecisions = {
  skipRows: number[];        // rowIndex values to skip (from preview)
  overwriteRows: number[];   // rowIndex values to UPDATE existing DB row
};

export type CommitRowResult =
  | { rowIndex: number; ok: true; action: 'inserted' | 'overwritten' | 'skipped'; orderId?: string }
  | { rowIndex: number; ok: false; action: 'insert' | 'overwrite'; error: string };

export type CommitResult =
  | { ok: true; batchId: string; committedCount: number; failedCount: number; results: CommitRowResult[] }
  | { ok: false; code: 'unauthorized' | 'validation' | 'server'; message: string };

export async function commitImportAction(
  formData: FormData,
