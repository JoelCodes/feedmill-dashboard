<objective>
Ship the four transition server actions that drive the directed state machine: `transitionToMixing`, `completeOrder`, `blockOrder`, `resumeFromBlocked`. All four follow the identical canonical action shape per RESEARCH.md §1, 7 and PATTERNS.md §src/actions/transitions.ts:

1. `'use server'` directive on line 1.
2. `await requireRole('mill_operator')` first line of body (AUTH-02, T-33-AuthZ).
3. Resolve `userId` from `auth()` for the audit trail.
4. Load current order state via `db.select().from(productionOrders).where(eq(id, orderId))` (used for state-machine guard, not for `version` — `version` comes from the client-supplied parameter so the optimistic check works).
5. Reject not-found with `{ ok: false, code: 'not_found' }`.
6. Reject illegal from-state with `{ ok: false, code: 'validation', message }` (e.g., `transitionToMixing` against a Completed order).
7. Execute the optimistic-concurrency UPDATE: `db.update(productionOrders).set({ state: toState, version: version + 1 }).where(and(eq(id, orderId), eq(version, version))).returning({ id: productionOrders.id })`.
8. If `.returning().length === 0`, return the LOCKED conflict response `{ ok: false, code: 'conflict', message: 'Order was modified by another user. Please refresh.' }` (D-02, ROADMAP SC#2 line 122).
9. Insert one row into `orderEvents` with `fromState`, `toState`, `changedBy: userId`, `note` (null for most actions; `reason` for blockOrder).
10. `revalidateTag('production-orders')` (TRANS-07, STATE.md mutation invariant — call BEFORE return).
11. Return `{ ok: true }`.

Purpose: This plan closes ROADMAP SC#1 (every transition writes order_events row), SC#2 (locked conflict message text + version-column optimistic concurrency), SC#3 (Block + free-text reason + resume to Mixing or Pending), and SC#4 (revalidateTag mandatory). It is the largest plan in Phase 33 by behavior surface — but the four actions share so much structure that they fit in one file and one test file.

Output: One server-action source file (`src/actions/transitions.ts` with 4 named exports plus the `TransitionResult` type) and one test file. No UI work (Phase 34). No new schema. No `src/db/index.ts` modification.
