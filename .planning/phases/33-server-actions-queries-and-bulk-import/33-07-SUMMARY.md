## Decisions Made

- **Direct Drizzle vs action invocation:** Action functions call `requireRole('mill_operator')` → `auth()` from `@clerk/nextjs/server`, which requires a Clerk request context (HTTP headers/cookies). In a Node CLI script there is no request context and `auth()` throws. The gap (GAP-03) is about Neon HTTP driver behavior, not authz. Direct Drizzle calls exercise the identical SQL generated by the action and fully satisfy the gap. Auth contract is covered by unit tests (TRANS-13, 33-06-SUMMARY.md). (T-33-07-NoAuth — accepted.)
- **Sentinel-keyed cleanup:** The `'harness-xlsx-test'` sentinel on all inserted rows ensures cleanup never touches production data (seed rows use `'system-seed'`; user rows use Clerk IDs).
- **dotenv first:** `loadDotenv({ path: '.env.local' })` is the first executable statement, before any import that transitively pulls in `src/db/index.ts`, which throws if `DATABASE_URL` is undefined.
- **xlsxSchema copy-paste:** The harness replicates the private `xlsxSchema` from `src/actions/import.ts` verbatim. A code comment marks it MUST MATCH the action to catch future drift.
