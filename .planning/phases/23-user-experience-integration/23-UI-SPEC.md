## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| space-1 | 4px | Icon gaps, inline padding |
| space-2 | 8px | Compact element spacing |
| space-3 | 12px | Form field padding |
| space-4 | 16px | Default element spacing |
| space-5 | 24px | Section padding |
| space-6 | 32px | Layout gaps, card padding |
| space-8 | 48px | Major section breaks |
| space-10 | 64px | Page-level spacing |

Exceptions: none (32px circular avatar uses space-6 base, within 8-point scale)

**Source:** `src/app/globals.css` lines 73-81, established in v1.3 design system

## Component Inventory

### UserButton (Clerk prebuilt)
- **Size:** 32px circular avatar
- **Position:** Header right section, after notification bell
- **States:** 3 states designed in Phase 22
  1. Default: Avatar with initial or photo
  2. Hover: Subtle background highlight
  3. Dropdown open: Menu with name/email + Sign Out

### Loading Skeleton
- **Size:** 32px circular placeholder
- **Color:** var(--divider) with subtle opacity
- **Pattern:** Matches existing OrdersTable skeleton approach
- **Duration:** Shows until Clerk auth state loads

### Dropdown Menu
- **Container:** var(--bg-card) background, var(--shadow-card) shadow
- **Border:** 1px solid var(--divider)
- **Radius:** var(--radius-lg) 12px
- **Padding:** var(--space-4) 16px
- **Contents:**
  1. User name/email display (top)
  2. Divider (var(--divider) 1px)
  3. "Sign Out" menu item (bottom)

**Source:** Phase 22 design in `designs/page-layout.pen`, 23-CONTEXT.md D-01 through D-07

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | UserButton is focusable, Enter/Space opens dropdown |
| Screen reader | aria-label="User menu" on button, aria-expanded state |
| Focus indicator | 2px outline var(--primary) on focus |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 minimum) |
| Touch target | 32px avatar meets 24px minimum (exceeds WCAG) |

**Source:** Existing WCAG 2.1 AA compliance from v1.3 (Phase 19), jsx-a11y rules

## Theme Support

UserButton respects current theme (light/dark) via:
1. CSS variable references in `clerkAppearance` config
2. Automatic token switching when `.dark` class applied to root
3. No hardcoded colors in Clerk appearance config

**Light mode:**
- Avatar bg: #4fd1c5 (teal primary)
- Dropdown bg: #ffffff (white card)
- Text: #2d3748 (dark gray)

**Dark mode:**
- Avatar bg: #63b3ed (blue primary)
- Dropdown bg: #2d3748 (dark gray card)
- Text: #e2e8f0 (light gray)

**Source:** 23-CONTEXT.md D-03, UX-03 requirement, `src/lib/clerk-theme.ts` lines 22-258
