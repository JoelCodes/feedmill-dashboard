<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Clerk SDK to Dashboard | Auth state flows from Clerk servers; dashboard trusts Clerk session validation |
| UserButton to Clerk API | Sign-out action calls Clerk API to invalidate session server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Spoofing | UserButton display | mitigate | Clerk validates session server-side before returning user data; no client-side user data storage |
| T-23-02 | Tampering | Sign-out redirect | accept | afterSignOutUrl is client-side preference; middleware protects routes regardless of redirect target |
| T-23-03 | Information Disclosure | User name/email | accept | User's own data displayed to authenticated user; no sensitive data exposure |
| T-23-04 | Denial of Service | ClerkLoaded wrapper | mitigate | Skeleton fallback ensures UI remains usable if Clerk SDK slow to initialize |
</threat_model>
