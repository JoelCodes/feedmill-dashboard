   - Sidebar from '@/components/Sidebar'

   - useLocalStorage from '@/hooks/useLocalStorage'
   - UserPreferences, defaultPreferences from '@/types/settings'
   - useState from 'react'

3. Component structure:
   - Use same layout pattern as orders/page.tsx (flex h-screen with Sidebar and main)
   - Header component (will show "Settings" title automatically via pathname)
   - Content area: max-w-2xl (640px per UI-SPEC), mx-auto for centering

4. State management:
   - const [savedPreferences, setSavedPreferences] = useLocalStorage<UserPreferences>('user-preferences', defaultPreferences)
   - const [formState, setFormState] = useState<UserPreferences>(savedPreferences)
   - Track unsaved changes: const hasChanges = JSON.stringify(formState) !== JSON.stringify(savedPreferences)

5. Notification Preferences section (per UI-SPEC):
   - Heading: "Notification Preferences" (text-lg font-bold mb-4)
   - Section spacing: mb-6
   - Three toggle checkboxes:
     - "Order Status Updates" -> formState.notifications.orderStatus
     - "Alerts" -> formState.notifications.alerts
     - "System Messages" -> formState.notifications.system
   - Each: label with flex items-center gap-3, checkbox input

6. Display Settings section (per UI-SPEC):
   - Heading: "Display Settings" (text-lg font-bold mb-4)
   - Theme select:
     - Label: "Theme"
     - Options: "Light" (value="light"), "Dark" (value="dark")
   - Density select:
     - Label: "Display Density"
     - Options: "Comfortable" (value="comfortable"), "Compact" (value="compact")

7. Save button:
   - Text: "Save Preferences" (per UI-SPEC copywriting)
   - Style: Primary button (bg-primary text-white px-4 py-2 rounded-lg)
   - onClick: setSavedPreferences(formState)
   - Disable or style differently when !hasChanges (optional enhancement)

8. Update handlers:
   - updateNotificationSetting(key, value) for checkboxes
   - updateTheme(value) for theme select
   - updateDensity(value) for density select
   - All update formState, not savedPreferences directly

9. Note per D-07 and CONTEXT.md: Theme/density visual changes are out of scope for Phase 5. The preferences persist to localStorage but do not yet apply visual changes to the UI. This is documented in UI-SPEC: "Note: Theme/density visual changes are out of scope - preferences persist only"
