2. **Auto-select first visible when current selection filtered out:**
   Add useEffect that selects first visible order when:
   - `validSelectedId` is null (current selection not in filtered list)
   - `selectedOrderId` is not null (had a selection before)
   - `filteredOrders.length > 0`

3. **Wrap onSelectOrder in useCallback** to avoid infinite loops in useEffect dependencies.

Pattern from research:
```typescript
// Auto-select first on load
useEffect(() => {
  if (!selectedOrderId && filteredOrders.length > 0) {
    onSelectOrder(filteredOrders[0].id);
  }
}, [selectedOrderId, filteredOrders, onSelectOrder]);

// Auto-select first if current selection filtered out
useEffect(() => {
  if (!validSelectedId && selectedOrderId && filteredOrders.length > 0) {
    onSelectOrder(filteredOrders[0].id);
  }
}, [validSelectedId, filteredOrders, selectedOrderId, onSelectOrder]);
```

Note: These effects run after initial data loads because `filteredOrders` starts empty until `getOrders()` resolves.
