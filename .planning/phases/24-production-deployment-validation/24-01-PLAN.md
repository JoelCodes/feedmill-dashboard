    User must configure Vercel environment variables via Dashboard.

    Navigate to: Vercel Dashboard > Project > Settings > Environment Variables

    **Production scope only (per D-05, D-06):**

    1. Add NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY:
       - Value: pk_live_* (copy from Clerk Dashboard > Production > API Keys)
       - Environments: Check ONLY "Production" (uncheck Preview, Development)

    2. Add CLERK_SECRET_KEY:
       - Value: sk_live_* (copy from Clerk Dashboard > Production > API Keys)
       - Environments: Check ONLY "Production" (uncheck Preview, Development)

    **Preview scope only (per D-05):**

    3. Add NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY:
       - Value: pk_test_* (copy from Clerk Dashboard > Development > API Keys, or use value from .env.local)
       - Environments: Check ONLY "Preview" (uncheck Production, Development)

    4. Add CLERK_SECRET_KEY:
       - Value: sk_test_* (copy from Clerk Dashboard > Development > API Keys, or use value from .env.local)
       - Environments: Check ONLY "Preview" (uncheck Production, Development)

    **Both Production and Preview:**

    5. Add NEXT_PUBLIC_CLERK_SIGN_IN_URL:
       - Value: /sign-in
       - Environments: Check "Production" AND "Preview"

    6. Add NEXT_PUBLIC_CLERK_SIGN_UP_URL:
       - Value: /sign-up
       - Environments: Check "Production" AND "Preview"

    After adding all variables, trigger a new deployment (push empty commit or redeploy via Dashboard).
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vercel Dashboard | Admin access to environment variables containing Clerk secrets |
| Clerk Dashboard | Admin access to production instance configuration and user data |
| Browser to Vercel | Public internet traffic to production deployment |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Information Disclosure | Vercel env vars | mitigate | Vercel auto-redacts secrets in build logs; never log CLERK_SECRET_KEY; verify no "All" scope exposes prod keys to dev |
| T-24-02 | Elevation of Privilege | Environment scoping | mitigate | Per D-05: Production keys scoped ONLY to Production; Preview keys scoped ONLY to Preview; verify in Dashboard before deploy |
| T-24-03 | Spoofing | Domain verification | mitigate | Per D-08: Vercel subdomain added to Clerk allowed origins; Clerk rejects requests from non-verified domains |
| T-24-04 | Information Disclosure | Test user credentials | mitigate | Per D-10: Credentials stored in GitHub Secrets only; never committed to repo; +clerk_test suppresses email trails |
</threat_model>
