## Spacing Scale

Declared values (all multiples of 4, sourced from `globals.css` `--space-*` tokens):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-1`) | Icon gaps, inline badge padding |
| sm | 8px (`--space-2`) | Compact element spacing, badge px |
| md | 16px (`--space-4`) | Default element spacing, card padding |
| lg | 24px (`--space-5`) | Section padding, column gap |
| xl | 32px (`--space-6`) | Layout gaps between KPI strip zones |
| 2xl | 48px (`--space-8`) | Major section breaks (between KPI strip and columns, between columns and KpiSection) |
| 3xl | 64px (`--space-10`) | Page-level vertical spacing |

Exceptions:
- KPI card inner padding: `px-[21px] py-[18px]` (`--card-padding-lg: 1.3125rem` = 21px) — matches existing `KPICard.tsx` prior art and `Card.Content` pattern.
- Column header KPI-03 strip: `mt-1` (4px) below the column title — matches existing `MillColumn.tsx` header spacing.
- Exception list table rows: `py-3` (12px) vertical padding — `--space-3` token value.

## Typography

All sizes use existing CSS custom-property tokens from `globals.css`:

| Role | Size | Weight | Line Height | Token |
|------|------|--------|-------------|-------|
| KPI card value (stat) | 22px | 700 (bold) | 1.2 | `--fs-22` |
| KPI card label / column header strip text | 11px | 600 (semibold) | 1.4 | `--fs-11` |

Weights used: **regular (400)** and **bold/semibold (600 / 700)**. The KPI stat value uses 700 for visual hierarchy; all labels use 600. Running body text uses 400. This matches the existing `KPICard.tsx` prior art (`font-bold` on value, `font-bold` on label) and `ProductionCard.tsx` (`font-bold` on customer name, `font-semibold` on order number).

Source: `globals.css` typography tokens; `KPICard.tsx` lines 62-70; `ProductionCard.tsx` lines 112-128.

## Component Inventory

### New Components (Phase 35)

#### `KpiCard.tsx`
Built on the existing `Card` primitive (`src/components/ui/Card.tsx`).

**Prop shape:**
```
label: string
value: string          // formatted aggregate (e.g., "18,400 lbs", "62%")
unit?: string          // displayed inline after value in smaller size
subValue?: string      // secondary line below value
footnote?: string      // small muted text at bottom (used for "N uncategorized" — KPI-05 D-12)
icon?: LucideIcon      // optional Lucide icon displayed in accent-colored container (right side)
```

**Visual structure:**
- Outer: `Card` default variant (`bg-[var(--bg-card)] border border-[var(--divider)] rounded-[var(--radius-lg)]`)
- Inner padding: `px-[21px] py-[18px]` (matches existing `KPICard.tsx` prior art)
- Left column: `label` at `--fs-11` semibold muted, `value` at `--fs-22` bold primary, `unit` inline at `text-sm` muted
- Optional `subValue` at `text-sm` regular muted, below value
- Optional `footnote` at `--fs-11` muted, bottom of card
- Right column (when `icon` present): `h-11 w-11` rounded-xl container filled `bg-[var(--primary)]` with `Icon className="h-5 w-5 text-white"`
- No `changeType` or delta — Phase 35 KPI cards show actuals only (OQ-1: no targets)

#### `KpiStrip.tsx`
Horizontal scrollable row of `KpiCard` instances.

**Layout:** `flex flex-row gap-4 w-full overflow-x-auto pb-1` — single row, no wrapping on desktop; overflow-x scroll on narrow viewports rather than wrapping.

**KPI cards in strip (left to right):**
1. Mill-wide tons completed today (KPI-01) — `label: "Completed Today"`, `value: "{N} lbs"`, `icon: Wheat`
2. Premix line tons today (KPI-02a) — `label: "Premix Today"`, `value: "{N} lbs"`, no icon
3. Excel line tons today (KPI-02b) — `label: "Excel Today"`, `value: "{N} lbs"`, no icon
4. CGM line tons today (KPI-02c) — `label: "CGM Today"`, `value: "{N} lbs"`, no icon
5. Pending backlog (KPI-04) — `label: "Pending Backlog"`, `value: "{N} orders"`, `subValue: "{N} lbs total"`, `icon: ClipboardList`
6. Formula mix (KPI-05) — three inline percentage values (Pellet / Mash / Crumble); if `footnote` has uncategorized count, render it; `icon: Activity` — rendered as a `KpiCard` with `subValue` per bucket or as the `KpiCard`'s `value` showing the dominant bucket

**KPI-02 rendering decision:** Per-line tons (KPI-02a/b/c) render as 3 separate compact `KpiCard` instances with `flex-shrink-0 min-w-[120px]` rather than as sub-cards inside KPI-01. This maximizes scannability of per-line values without nested layout complexity. Source: CONTEXT.md "Claude's Discretion" item on KPI-02 arrangement.

**KPI-05 card layout:** Single card. `value` shows dominant bucket percentage + name (e.g., "58% Pellet"). `subValue` shows the remaining two (e.g., "32% Mash · 10% Crumble"). `footnote` shows "Excludes N uncategorized orders" when count > 0 (D-12).

**Skeleton (`KpiStripSkeleton`):**
- 6 inline rectangular placeholders: `h-[88px] w-[140px] animate-pulse rounded-[var(--radius-lg)] bg-[var(--divider)]` with `gap-4 flex flex-row`
- Matches the KpiStrip's flex row layout

#### `KpiCard` — KPI-03 column header inline (not a separate component)
Not a standalone component — rendered inline in `MillColumn.tsx` header via a `<p>` extension.

**Format:** `{N} orders — {completedLbs} / {totalLbs} lbs`
- `{N} orders` at `--fs-11` semibold `var(--text-primary)`
- ` — ` separator at `--fs-11` muted
- `{completedLbs} / {totalLbs} lbs` at `--fs-11` muted
- Rendered as the second line of the existing `MillColumn` header, `mt-1` below the column name `h2`
- Replaces the existing `formatWeight(completed) / formatWeight(total) lbs` line (same data, same position, new count prefix)

Source: `MillColumn.tsx` lines 113-118 (existing header text pattern).

#### `SevenDayTrendChart.tsx`
Hand-rolled inline SVG. No chart library. (D-13 / CONTEXT.md)

**Props:** `data: Array<{ date: string; completedLbs: number }>` (length 0..7)

**SVG dimensions:** `width="100%"` viewBox `0 0 420 160` (responsive via `preserveAspectRatio="xMidYMid meet"`). 7 bars at 48px wide with 8px gap; bar height proportional to max value in dataset; minimum bar height 4px for non-zero values.

**Bar fill:** `var(--primary)` (#4fd1c5 teal). Opacity 80% on past days, 100% on today's bar.

**Axis labels:** X-axis day labels below each bar — `--fs-11` `var(--text-muted)` — formatted as "Mon", "Tue", etc. (3-letter weekday abbreviation from the `date` string). Y-axis: no explicit Y-axis line; max value label at top-right of chart in `--fs-11` muted.

**Empty state (fewer than 7 days of data):** Render the chart card but replace SVG with centered text: heading "Not enough data yet" at `text-sm` semibold `var(--text-primary)` + body "Check back after 7 days of production" at `--fs-13` `var(--text-muted)`. Card dimensions preserved so layout does not collapse.

**Card wrapper:** `Card` default variant, `p-4`, with a section heading "7-Day Volume Trend" at `text-sm` semibold `var(--text-medium)` above the SVG.

#### `BlockedExceptionList.tsx`
Sortable table of blocked orders. Distinct from `BlockedAlertBand` (D-10).

**Props:** `orders: Array<BlockedOrderWithDwell>` where `dwell` is a duration string computed server-side.

**Layout:** `Card` default variant. Header row: "Blocked Orders" heading at `text-sm` semibold `var(--text-medium)` + sort affordance label "Sorted by dwell time" at `--fs-11` muted (right-aligned). KPI-07 ships with default dwell-time sort only — no click-to-sort column headers in v2.0 (deferred per CONTEXT.md).

**Table columns (in order):**
1. Order # — `--fs-11` semibold muted; `min-w-[80px]`
2. Customer — `text-sm` regular `var(--text-primary)`; `flex-1`
3. Mill Line — `--fs-11` semibold; `min-w-[72px]`; text matches `STATE_COLORS` `--status-{state}-header` for the mill-line display (no color — plain muted)
4. Blocked For (dwell time) — `text-sm` semibold `var(--text-primary)`; `min-w-[80px]`
5. Overdue badge (KPI-08) — conditionally rendered; see badge spec below; `min-w-[64px]`

**Row interaction:** Entire row is `role="button"` + `tabIndex={0}` + Enter/Space keyboard handler. Click sets `?order=<id>` via `useOrderQuery()` (reusing Phase 34 pattern). Hover: `hover:bg-[var(--pending-light)]`. Active: `active:opacity-90`.

**Empty state (no blocked orders):** Card remains visible. Content: centered "No blocked orders" at `text-sm` muted — no icon. Minimum card height `min-h-[64px]` to prevent layout collapse.

**Dwell time format:** Server-computed duration string, formatted as:
- Under 1 hour: `"Xm"` (e.g., `"42m"`)
- 1+ hours: `"Xh Ym"` (e.g., `"2h 14m"`)
- 24+ hours: `"Xd Yh"` (e.g., `"1d 3h"`)
Source: CONTEXT.md D-03 — "Display formatting is a UI concern, locked via UI-SPEC."

**Overdue badge (KPI-08):**
- Renders only when `earlyDeliveryDate` is non-null AND `earlyDeliveryDate < today` AND `state !== 'Completed'`
- Visual: inline badge `bg-[var(--warning-light)] text-[var(--warning)] border border-[var(--warning)] rounded-[var(--radius-sm)] px-2 py-0.5 text-[var(--fs-11)] font-semibold`
- Label: `"Overdue"`
- Accessible: `role="status"` `aria-label="Order past early delivery date"`
- Reuses `StatusBadge` color palette approach but does NOT extend `StatusBadge.tsx` directly — it is an inline span in `BlockedExceptionList` and `ProductionCard` (see below)

**Overdue badge in `ProductionCard` (KPI-08 in list view):**
- Rendered below the delivery time line (`<p>` with `mt-1`)
- Same visual spec as above

#### `KpiSection.tsx`
Full-width container below the 3 columns.

**Layout decision:** Side-by-side on md+ viewports, stacked on mobile.
- `flex flex-col gap-5 md:flex-row md:gap-6`
- `SevenDayTrendChart` takes `flex-1` (wider)
- `BlockedExceptionList` takes `w-full md:w-[380px] flex-shrink-0` (fixed right panel)

This arrangement puts the trend chart on the left (more visual weight, wider) and the exception list on the right (a sidebar panel feel). On mobile they stack with the exception list below the chart.

**Section heading:** No outer heading for `KpiSection` itself — each child card has its own header.

**Skeleton (`KpiSectionSkeleton`):**
- Two side-by-side rectangles: `h-[200px] flex-1 animate-pulse rounded-[var(--radius-lg)] bg-[var(--divider)]` and `h-[200px] w-[380px] animate-pulse rounded-[var(--radius-lg)] bg-[var(--divider)]`

## Interaction Contract

### Read-only surfaces
All KPI components are read-only. No mutation affordances. No role gate required — all authenticated users see KPIs. (CONTEXT.md D-17)

### BlockedExceptionList row click
- Click or Enter/Space on any row sets `?order=<id>` (non-shallow via `useOrderQuery()`)
- Opens `ProductionDrawer` with `canEdit` determined by Phase 34/31 logic (mill_operator gate)
- Transition buttons hidden for non-operators — same as Phase 34 read-only pattern

### KpiStrip scrolling
- Horizontal scroll when content overflows on narrow viewports (no clip, no wrap)
- No keyboard focus trap — standard scroll container

### SevenDayTrendChart
- Static SVG — no hover tooltips, no click interaction in v2.0 (deferred per CONTEXT.md)
- Bars are decorative; `aria-hidden="true"` on SVG; chart card has `aria-label="7-day production volume trend"` on wrapper `section`

### Overdue badge in ProductionCard
- Rendered inline, not interactive
- `role="status"` accessible label

### Suspense + polling interaction
- `KpiStrip` wrapped in `<Suspense fallback={<KpiStripSkeleton />}>`
- `KpiSection` wrapped in `<Suspense fallback={<KpiSectionSkeleton />}>`
- On `router.refresh()` (polling or manual), Suspense fallbacks re-show briefly during RSC re-render
- If stale-values behavior is needed after UAT feedback, apply `useTransition` + isPending dimming pattern (Phase 34 D-23 reference) — planner decides based on smoke test results

## Accessibility

- All KPI cards: semantic `<article>` or `<div role="region" aria-label="{card label}">` wrapper
- SevenDayTrendChart SVG: `aria-hidden="true"` on SVG element; wrapping div has `role="img" aria-label="7-day production volume trend bar chart"`
