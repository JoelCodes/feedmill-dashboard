```typescript
export type ActivityEventType =
  | 'order_placed'
  | 'production_started'
  | 'ready'
  | 'out_for_delivery'
  | 'delivered'
  | 'delivery_completed'
  | 'bin_alert_low'
  | 'bin_alert_critical';

export interface ActivityEvent {
  id: string;
  customerId: string;
  type: ActivityEventType;
  timestamp: Date;
  title: string;
  description: string;
  // Order-specific fields (optional)
  orderId?: string;
  orderQuantity?: number;
  orderProduct?: string;
  orderStatus?: string;
  // Bin-specific fields (optional)
  binId?: string;
  binLocationCode?: string;
  binFillPercentage?: number;
}
```

2. Create `src/services/activity.ts`:
- Import mockOrders, mockBins from mockData.ts
- Implement `getActivityEvents(customerId: string): Promise<ActivityEvent[]>`
- Generate events from orders per D-03 (one event per order lifecycle stage based on current status):
  - All orders: order_placed event (use createdAt)
  - Status Producing+: production_started event
  - Status Ready+: ready event
  - Status In Transit+: out_for_delivery event
  - Status Complete: delivered event
- Generate bin alert events from bins where alertLevel is "low" or "critical" (use lastUpdated)
- Sort all events by timestamp descending
- Use title/description templates from UI-SPEC Copywriting Contract

3. Title templates per UI-SPEC:

- delivered: "Order #{documentNumber} Delivered"
- bin_alert_low: "Low Feed Alert - Bin {locationCode}"

4. Description templates per UI-SPEC:
