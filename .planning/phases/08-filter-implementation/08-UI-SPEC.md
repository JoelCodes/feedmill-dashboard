## Interaction Patterns

### Filter Toggle Behavior

**Pattern:** Multi-select toggle with Set state management

**User flow:**
1. User lands on mill production page → all filters inactive → all cards visible
2. User clicks "Completed" pill → pill turns blue → only Completed cards visible
3. User clicks "Mixing" pill → both pills blue → Completed + Mixing cards visible
4. User clicks "Completed" again → only Mixing pill blue → only Mixing cards visible
5. User clicks "Mixing" again → no pills blue → all cards visible again

**Implementation:**
- State: `useState<Set<ProductionState>>(new Set())`
- Toggle: `Set.has(state) ? Set.delete(state) : Set.add(state)` (with new Set for immutability)
- Filter logic: `activeStates.size === 0 ? orders : orders.filter(o => activeStates.has(o.state))`

**Visual feedback:**
- Active pill: Primary blue background (#4fd1c5), white text, white dot, white/20% count badge
- Inactive pill: State-specific light background, dark text, colored dot, translucent count badge
- Hover: 90% opacity on all pills (hover:opacity-90)
- Non-matching cards: Hidden completely (display: none), not dimmed

**Source:** Interaction pattern from 08-CONTEXT.md D-03, D-04, D-05. Implementation pattern from RESEARCH.md "Pattern 1: Set-Based Multi-Select Toggle" and existing OrdersTable.tsx (lines 40-50, 68-79).

### Count Badge Behavior

**Pattern:** Static count (unaffected by filters)

**Rule:** Count badges always show total orders per status from full dataset, regardless of which filters are active.

**Example:**
- Total dataset: 6 Completed, 4 Mixing, 2 Blocked, 3 Pending
- User selects "Mixing" filter
- Pills show: Completed (6), Mixing (4), Blocked (2), Pending (3) ← counts unchanged
- Cards show: Only the 4 Mixing cards visible

**Implementation:**
```typescript
const stateCounts = useMemo(() => {
  return STATE_ORDER.reduce((acc, state) => {
    acc[state] = orders.filter(o => o.state === state).length;
    return acc;
  }, {} as Record<ProductionState, number>);
}, [orders]); // Depends ONLY on orders, NOT on activeStates
```

**Source:** 08-CONTEXT.md D-06, D-07. RESEARCH.md "Pattern 3: Static Count Badges" and "Pitfall 4: Count Badges Changing with Filters".

| Requirement | Implementation |
|-------------|----------------|

### Key Constraints

1. **Extract, don't duplicate:** FilterPill must be extracted from OrdersTable.tsx to src/components/FilterPill.tsx, then OrdersTable refactored to import shared component (per D-01)
2. **Generic props:** FilterPill must NOT hardcode ProductionState or OrderStatus types — accept generic color prop (per D-02)
3. **Complete hiding:** Non-matching cards hidden (display: none), not dimmed with opacity (per D-03, matching orders page pattern)
4. **Static counts:** Count computation must depend only on `orders` array, NOT on `activeStates` or `filteredOrders` (per D-06, D-07)
5. **Immutable Set updates:** Always create `new Set(prev)` before modifying — direct mutation breaks React change detection (RESEARCH.md Pitfall 1)
