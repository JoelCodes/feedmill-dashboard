|------|------|--------|-------------|

## Component Inventory

Components to design in .pen files:

**customers.pen (Customer List View):**

1. **CustomerTable**
   - Table rows layout matching OrdersTable pattern (D-01)
   - Columns: Name + Status (D-02)
   - Combined status indicator with stacked icons (D-03)
   - Row click navigates to detail page
   - Hover state: bg-gray-50
   - Selected state: not applicable (navigation triggers immediately)

2. **CustomerSearchBox**
   - Matches Header search styling (D-16)
   - Placeholder: "Search customers by name..."
   - Icon: Search from lucide-react
   - Real-time filter behavior

3. **StatusIndicator** (stacked icons)
   - Order count badge (Package icon + number)
   - Changes dot (red dot if present)
   - Bin alert icon (AlertTriangle, color: yellow/red based on threshold)
   - Icons from lucide-react (D-14)

**customer-detail.pen (Customer Detail Page):**

4. **CustomerDetailHeader**
   - Fixed position header (D-05)
   - Full contact card: name, location, contact info, delivery preferences (D-06)
   - No back button (browser back only per D-08)

5. **CustomerSummaryStats**
   - Total orders count
   - Active bins count
   - Recent activity count (last 30 days)
   - Card layout with subtle shadows

6. **BinGaugeRow** (Bin Visualization)
   - Vertical tank gauge (D-09)
   - Fill grows bottom-to-top
   - Color zones: green (normal), yellow (low), red (critical) (D-10)
   - Compact row layout (D-11)
   - Location code + feed type labels near gauge (D-12)
   - Percentage shown inside gauge

7. **ActivityTimeline**
   - Extends TimelineItem pattern from OrderDetails (D-15)
   - Event types: orders, deliveries, bin alerts
   - Chronological order (newest first)
   - Expand/collapse interaction
   - Order events link to /orders page

**Source**: Decisions D-01 through D-16 from 10-CONTEXT.md, canonical references to OrdersTable.tsx and OrderDetails.tsx

## Interaction States

| Component | States | Behavior |
|-----------|--------|----------|
| CustomerTable row | default, hover, (no selected) | hover: bg-gray-50, click navigates to /customers/[id] |
| CustomerSearchBox | default, focus | focus: border-primary ring-primary |
| BinGauge | default, hover | hover shows full metadata tooltip |
| TimelineItem | collapsed, expanded | click toggles expanded state showing full details |
| Order link in timeline | default, hover | hover: underline, click navigates to /orders?selected=[orderId] |

**Source**: Inferred from OrdersTable interaction patterns and decisions D-01, D-03, D-07

## Visual Focal Points

**Customer list view (customers.pen):**
- **Primary focal point**: CustomerSearchBox — positioned at top of list, accent color on focus state draws eye first
- **Secondary focal point**: First CustomerTable row — high-contrast status indicators create visual weight

**Customer detail view (customer-detail.pen):**
- **Primary focal point**: CustomerDetailHeader — fixed position at top, contains customer name at Display size (20px bold)
- **Secondary focal point**: BinGaugeRow section — color-coded gauges (green/yellow/red) create immediate visual hierarchy below header
- **Tertiary focal point**: ActivityTimeline — chronological layout guides eye downward after header and bins

**Source**: Recommended by gsd-ui-checker Dimension 2 (Visuals)
