## Routing Contract

**Phase 13 routing implementation:**

| Route | Component | Data Loading |
|-------|-----------|--------------|
| `/customers/[id]` | CustomerDetailPage | Server Component: fetch customer by ID + stats |
| `/customers/[id]?tab=orders` | CustomerDetailPage (Orders tab) | Deferred to Phase 14 (timeline) |
| `/customers/[id]?tab=bins` | CustomerDetailPage (Bins tab) | Deferred to Phase 15 (bin visualization) |

**Navigation patterns:**

1. **From customer list to detail:**
   - Customer row click → `router.push('/customers/${customer.id}')`
   - Source: Phase 12 customer list page

2. **From detail to orders page:**
   - Order link click → `router.push('/orders?selected=${orderId}')`
   - Source: REQUIREMENTS.md CDET-03

3. **From detail back to list:**
   - Browser back only (no explicit back button per Phase 10 Decision D-08)
   - Fallback: clicking "Customers" in sidebar nav

**Data fetching:**
```typescript
// Server Component pattern
async function CustomerDetailPage({ params }: { params: { id: string } }) {
  const customer = await customersService.getCustomerById(params.id);
  const stats = await customersService.getCustomerStats(params.id);

  if (!customer) {
    notFound(); // Next.js 404
  }

  return <CustomerDetailHeader customer={customer} stats={stats} />;
}
```

**Source**: REQUIREMENTS.md CDET-01, CDET-03, Phase 10 UI-SPEC.md Decision D-08
