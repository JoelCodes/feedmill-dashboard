## Interaction States

### Navigation Item States

| State | Visual Treatment | Trigger |
|-------|-----------------|---------|
| Default | White icon background, gray text (`var(--text-secondary)`) | Item not active |
| Active | Primary color icon background (`var(--primary)`), white icon, dark text (`var(--text-primary)`), elevated card shadow | Current route matches item href (prefix matching) |
| Hover | Existing styles (no change required per existing implementation) | Mouse over |

**Active Detection Logic**:
- Use Next.js `usePathname()` hook
- Prefix match: `/orders/123` activates "Orders" nav item
- Root path `/` activates "Dashboard" nav item
- `/mill-production` activates "Production" nav item

**Source**: Existing NavItem active styling in Sidebar.tsx lines 91-114. CONTEXT.md D-01, D-02 specify auto-detection approach.
