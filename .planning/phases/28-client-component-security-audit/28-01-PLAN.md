<jest_hoisting_constraint>
**CRITICAL Jest hoisting rule:** `jest.mock()` calls are hoisted by Jest to the top of the file. They cannot reference variables declared with `const`/`let` in the same file (would produce ReferenceError at hoist time).

**Implication for this fixture:** The fixture CANNOT export a "drop-in setup function" that calls `jest.mock()` on behalf of consumers. Consumers must call `jest.mock()` themselves and pass the fixture's factory as the second argument. The fixture exports the FACTORY and the helpers; consumers wire them up. See task 1 for the exact API shape.
</jest_hoisting_constraint>
