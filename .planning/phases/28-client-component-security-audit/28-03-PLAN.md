<refactor_shape>
**Critical structural decision:** The current page has TWO components in one file — `OrdersPage` (the default export) and `OrdersContent` (inner, owns `useSearchParams` and `selectedOrderId` state). After refactor:
- `OrdersPage` becomes an **async RSC** (top-of-file: remove `'use client'`; mark function `async`).
- `OrdersContent` remains a **client component** but must move to its own file `src/components/OrdersTableContent.tsx` (with `'use client'` directive) so the RSC `OrdersPage` can import it cleanly without TYPE WARNINGS about mixing server and client in the same file.
- `OrdersContent` accepts `orders: Order[]` as a new prop and forwards it to `<OrdersTable orders={orders} ...>`.

This extraction is unavoidable: a `'use client'` directive applies to the entire file, so an RSC and a client subcomponent cannot coexist in the same file.

**Filename choice:** `OrdersTableContent.tsx` (per RESEARCH.md Code Example A). Self-locates next to `OrdersTable.tsx` in `src/components/`, signals "wraps OrdersTable with URL-param state."
</refactor_shape>
