CRITICAL: Follow the exact pattern from RESEARCH.md to avoid "auth() was called but Clerk can't detect usage of clerkMiddleware()" errors.

File content:
```typescript
import { clerkMiddleware, createRouteMatcher } from "@clerk/nextjs/server";

/**
 * Routes that are publicly accessible without authentication.
 * All other routes are protected by default.
 */
const isPublicRoute = createRouteMatcher([
  "/sign-in(.*)",
  "/sign-up(.*)",
]);

export default clerkMiddleware(async (auth, request) => {
  // Protect all routes except public ones
  if (!isPublicRoute(request)) {
    await auth.protect();
  }
});

/**
 * Middleware matcher configuration.
 * Uses broad matcher to ensure middleware runs on all routes.
 * Excludes static files and Next.js internals.
 */
export const config = {
  matcher: [
    // Skip Next.js internals and static files
    "/((?!_next|[^?]*\\.(?:html?|css|js(?!on)|jpe?g|webp|png|gif|svg|ttf|woff2?|ico|csv|docx?|xlsx?|zip|webmanifest)).*)",
    // Always run for API routes
    "/(api|trpc)(.*)",
  ],
};
```

Key decisions (per RESEARCH.md pitfalls):
- Uses async `auth.protect()` (Clerk v6+ async requirement)
- Broad matcher pattern ensures middleware catches all routes
- Public routes include `/sign-in(.*)` for catch-all route support (D-04)
- `/sign-up(.*)` included for future sign-up page (though deferred per D-05)
- No `/api/(.*)` in public routes - API routes require auth by default

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> middleware | All HTTP requests pass through middleware before reaching app |
| middleware -> Clerk API | Session validation requires Clerk API calls |
| browser -> env vars | NEXT_PUBLIC_ vars exposed to client-side JS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01 | Spoofing | middleware.ts | mitigate | Use auth.protect() to validate session tokens via Clerk |
| T-20-02 | Tampering | .env.local | mitigate | Gitignore .env.local; use .env.example for template only |
| T-20-03 | Information Disclosure | .env.local | mitigate | CLERK_SECRET_KEY has no NEXT_PUBLIC_ prefix - server-only |
| T-20-04 | Elevation of Privilege | middleware config | mitigate | Broad matcher ensures no routes bypass middleware |
| T-20-05 | Spoofing | CVE-2025-29927 | accept | Next.js 16.1.6 is above the fixed version (15.2.3) |
