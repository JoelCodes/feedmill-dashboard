<objective>
Extend the Book1.xlsx import pipeline to persist `early_delivery_date` end-to-end. The Phase 33 import already PARSES the `Early Delivery Date` XLSX column into `raw.deliveryDate` and converts it via `dateToIsoString()` (used today only for the display-text `deliveryTime` column). Phase 35 makes the same parsed value flow into the new `early_delivery_date` DB column on both the INSERT path (new orders) and the OVERWRITE path (duplicate decision === 'overwrite').

Purpose: KPI-08's overdue badge depends on `production_orders.early_delivery_date` being populated for ALL orders that originate from a Book1.xlsx import. Seed data (Plan 35-01 Task 3) provides dev-environment visibility, but real operator workflow imports XLSX files ŌĆö without this plan, every imported row would have a NULL `early_delivery_date` and the overdue badge would never fire.

Output: Zod schema gains `earlyDeliveryDate` field; `commitImportAction` writes the new column on both insert and update paths; new unit tests confirm Pitfall 7 (overwrite path) is closed; full Jest suite + tsc clean.
</objective>
