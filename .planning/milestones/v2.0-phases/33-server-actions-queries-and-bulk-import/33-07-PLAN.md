<objective>
Close VERIFICATION.md **GAP-03** (End-to-end XLSX import against live Neon dev DB) by adding a re-runnable scripted dev-DB harness that exercises both `previewImportAction` and `commitImportAction` against the real `@neondatabase/serverless` HTTP driver — the one path that unit tests with mocked `@/db` cannot reach.

The harness loads `example-data/Book1.xlsx` from disk, constructs a real `FormData` with a real `File` (Web API, available in Node 24 globally), calls `previewImportAction(formData)`, asserts the summary shape, then calls `commitImportAction(formData, decisions)` with empty skip/overwrite decisions, asserts the `CommitResult`, and verifies the live DB state delta:

1. **production_orders** row count increased by exactly `committedCount`.
2. **import_batches** got exactly one new row with `fileName === 'Book1.xlsx'`, `rowCount === committedCount`.

It cleans up after itself in a `finally` block so re-runs are deterministic. The cleanup uses a sentinel — every harness-inserted row has `created_by = 'harness-xlsx-test'` — but `created_by` is not a parameter accepted by `commitImportAction` (the action sets it from the Clerk userId). To use the sentinel as the cleanup key, the harness stubs `@clerk/nextjs/server`'s `auth()` and `@/lib/auth`'s `requireRole` via module-level mocks WITHOUT touching the action source.
