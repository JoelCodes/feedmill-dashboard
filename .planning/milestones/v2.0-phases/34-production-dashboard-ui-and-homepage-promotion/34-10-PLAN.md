<objective>
Close gap T10a (UAT.md: "Operator can transition a Pending order directly to Blocked").

Per the user's routing decision (UAT.md Gaps table, 2026-05-14): **amend D-11** to allow Block directly from Pending. This is NOT a code bug under the previous design contract — it is a deliberate design change. The user wants Block available from Pending without going through Mixing first (the previous spec required Pending → Mixing → Block).

Three load-bearing facts make this a small fix:
1. The server action `blockOrder(orderId, version, reason)` (`src/actions/transitions.ts:215`) ALREADY accepts `fromState='Pending'` — line 246 has `if (!['Pending', 'Mixing'].includes(order.state))`. No server-side change required.
2. `BlockReasonModal.tsx` is already reusable from any caller — it takes `orderId` + `version` and calls `blockOrder` directly. No modal change required.
3. The drawer (`ProductionDrawer.tsx:246-253`) renders `<TransitionButtons />` whenever `canEdit && order.state !== 'Completed'`. For Pending orders, the drawer is already mounted; only the `TransitionButtons` switch case needs to render the Block trigger alongside Start Mixing.

The fix is therefore: (a) update `TransitionButtons.tsx` Pending case to render both `StartMixingButton` AND `BlockOrderTrigger`, (b) amend D-11 in `PATTERNS.md` and `CONTEXT.md`, (c) extend `TransitionButtons.test.tsx` to assert Block button visible on Pending + clickable + invokes onBlockClick.

Purpose: Give the operator a one-click escape hatch from Pending to Blocked without the artificial Pending → Mixing → Block detour. Aligns the UI with the audit-trail-first design philosophy that already underpins D-11 (Mistakes are recoverable; the audit trail captures every transition).

Output:
- `TransitionButtons.tsx` Pending case mirrors the Mixing case's button pair (primary + destructive).
- `TransitionButtons.test.tsx` proves the new wiring (Block button visible on Pending, click fires onBlockClick callback).
- `PATTERNS.md` and `CONTEXT.md` D-11 amendments record the design change for future readers.
</objective>
