|----------|-------------|
| Client -> Middleware | Untrusted request crosses into server-side middleware |
| Middleware -> Clerk | Middleware reads session claims from Clerk-signed JWT |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-03 | Spoofing | Session claims | mitigate | Clerk validates JWT signature server-side; middleware only reads validated sessionClaims |
| T-25-04 | Tampering | Role claim | mitigate | publicMetadata is read-only from client, writable only via Clerk Backend API |
| T-25-05 | Elevation of Privilege | /demo/* access | mitigate | Middleware checks role BEFORE page renders; client cannot bypass server middleware |
| T-25-06 | Information Disclosure | Redirect behavior | accept | 307 redirect reveals route exists but no sensitive data; standard UX pattern |
