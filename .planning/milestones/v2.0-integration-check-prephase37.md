## Detailed Findings

### BLOCKER-01 — Build Fails: `BlockedAlertBand.tsx` startTransition void cast missing

**Affects:** All E2E flows — app cannot be built for production.

**File:** `src/components/BlockedAlertBand.tsx:44`

**Evidence:** `npm run build` output:
```
./src/components/BlockedAlertBand.tsx:44:48
Type error: Type 'Promise<URLSearchParams>' is not assignable to type 'VoidOrUndefinedOnly | Promise<VoidOrUndefinedOnly>'.
```

**Root cause:** `startTransition(() => setQuery({ order: order.id }))` passes `setQuery(...)` as an expression directly to the startTransition callback. `useQueryStates` (nuqs 2.8.9) returns `Promise<URLSearchParams>` from its setter. React's `startTransition` requires `() => void`. The expression form exposes the Promise return value; the block form (used correctly in `ProductionDrawer.tsx:133-135` and `ProductionDashboard.tsx:297-326`) does not.

**Fix:** Change line 44 to `startTransition(() => void setQuery({ order: order.id }))` — matching the pattern already used in `BlockedExceptionList.tsx:35`.

**Requirement mapping:** PROD-06 (blocked alert band) is the feature; the build failure blocks ALL requirements from reaching production.

**Classification: BLOCKER**
