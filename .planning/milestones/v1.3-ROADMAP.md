**Status:** SHIPPED 2026-05-09

## Overview

Establish a unified design system with tokens, components, and theming — then migrate existing pages to use it.

**Requirements**: FOUND-01, FOUND-02, FOUND-03, FOUND-04, DES-01, DES-02, DES-03

## Milestone Summary

**Key Decisions:**
- Use rules approach for jsx-a11y to avoid plugin conflict with eslint-config-next
- Configure all 31 jsx-a11y recommended rules at error severity for WCAG compliance
- Use color-mix(in_srgb,var(--token) N%,transparent) for opacity variants without dedicated tokens
- Two-tier token naming pattern: primitives in :root, semantic aliases in @theme inline
- CVA for type-safe variant definitions in all components

**Issues Resolved:**
- Dark mode flash on page load (via next-themes with suppressHydrationWarning)
- Hardcoded hex/px values eliminated from all pages
- Accessibility violations fixed across all components and pages

**Issues Deferred:**
- None

**Technical Debt Incurred:**
- FilterPill, Gauge, Timeline skip cn() utility (using manual Tailwind classes)
- StatusBadge uses STATUS_CONFIG object not CVA (maintains existing API)
