All 10 components have automated accessibility tests passing (Plan 02). README.md documentation is complete with accessibility notes for each component (Plan 03). Now manual VoiceOver testing is needed to verify screen reader announcements.
  </what-built>
  <how-to-verify>
**VoiceOver Testing Checklist**

1. **Enable VoiceOver:** Cmd+F5 (or System Settings > Accessibility > VoiceOver)
2. **Navigate with VO keys:** Ctrl+Option + Arrow keys

**Test each component:**

**Button (Orders page or Settings page)**
- [ ] Tab to button — VoiceOver announces button role and text
- [ ] Focus ring visible
- [ ] Enter/Space activates button
- [ ] Loading button announces "busy" state

**Input (Customers search)**
- [ ] Tab to input — announces label/placeholder
- [ ] Type text — VoiceOver reads input
- [ ] If error shown — error message announced

**Select (if present on any page)**
- [ ] Tab to select — announces label and current value
- [ ] Arrow keys change selection
- [ ] Selection change announced

**ThemeToggle (Settings page)**
- [ ] Tab to toggle — announces current theme
- [ ] Enter/Space toggles theme
- [ ] New theme state announced

**FilterPill (Orders or Mill Production)**
- [ ] Tab to pill — announces label and pressed state
- [ ] Toggle state change announced

**Card (Customers list)**
- [ ] If clickable — announces as button
- [ ] Tab-accessible with visible focus

**Timeline (Customer detail page)**
- [ ] Events navigable with Tab or VO keys
- [ ] Event content announced

**Gauge (Customer detail page)**
- [ ] Label and value accessible
- [ ] Status (normal/warning/critical) announced

**Testing URLs:**
- http://localhost:3000/orders (FilterPill, Button, StatusBadge)
- http://localhost:3000/customers (Input, Card)
- http://localhost:3000/customers/[any-id] (Timeline, Gauge)
- http://localhost:3000/settings (ThemeToggle, Button)

All components pass [jest-axe](https://github.com/nickcolley/jest-axe) automated accessibility checks. Tests are located in each component's `.test.tsx` file under the "Accessibility" describe block.

**Run accessibility tests:**
```bash

```

[eslint-plugin-jsx-a11y](https://github.com/jsx-eslint/eslint-plugin-jsx-a11y) is configured with recommended rules. Run linting:
```bash
npm run lint
```

### VoiceOver Manual Verification

Manual screen reader testing completed on [DATE] using macOS VoiceOver.

**Verified behaviors:**

| Component | Keyboard Navigation | Screen Reader Announcement | Focus Visible |
|-----------|--------------------|-----------------------------|---------------|
| Button | Enter/Space activates | Role and text announced | Yes |
| Card (clickable) | Tab-accessible, Enter activates | Announced as button | Yes |
| Input | Tab focuses, typing works | Label announced on focus | Yes |
