<task type="auto">
  <name>Task 1: Suppress tailwindcss config warning for Tailwind v4</name>
  <files>eslint.config.mjs</files>
  <action>
The eslint-plugin-tailwindcss tries to resolve tailwind.config.js but this project uses Tailwind v4 with CSS-based config in globals.css. The plugin emits "Cannot resolve default tailwindcss config path" for every file.

Current config has `config: null` in settings but this still triggers warnings. The solution is to tell the plugin there is no config file to resolve by using an empty config object or disabling config resolution entirely.

Update the tailwindcss plugin settings block in eslint.config.mjs:

```javascript
settings: {
  tailwindcss: {
    // Tailwind v4 uses CSS-based config, not JS config file
    // Set config to empty object to prevent resolution warnings
    config: {},
    skipClassAttribute: false,
    callees: ["classnames", "clsx", "ctl", "cn"],
  },
},
```

If that doesn't suppress the warning, try setting `config: "tailwind.config.js"` with an explicit path that will be ignored since we disable no-custom-classname anyway. Or use `config: false`.

Verify by running `npm run lint 2>&1 | grep -c "Cannot resolve default tailwindcss"` - should be 0.
  </action>
  <verify>npm run lint 2>&1 | grep -c "Cannot resolve default tailwindcss" returns 0</verify>
  <done>ESLint runs without "Cannot resolve default tailwindcss config path" warnings</done>
</task>

<task type="auto">
  <name>Task 2: Convert ESLint test file to ESM imports</name>
  <files>eslint-rules/no-hardcoded-values.eslint-test.js</files>
  <action>
The test file uses `require()` which violates @typescript-eslint/no-require-imports rule:
```javascript
const { RuleTester } = require("eslint");
const rule = require("./no-hardcoded-values");
```

Options:
1. Convert to ESM imports and rename to .mjs
2. Add eslint-disable comment for this specific file (test tooling file)

Since this is a test file for the ESLint rule itself and lives outside src/, the simplest fix is to add a file-level eslint-disable comment. The file is a Node.js script that runs via `node eslint-rules/no-hardcoded-values.eslint-test.js`.

Add at top of file:
```javascript
/* eslint-disable @typescript-eslint/no-require-imports */
```

Or if converting to ESM is preferred, rename to .mjs and use:
```javascript
import { RuleTester } from "eslint";
import rule from "./no-hardcoded-values.js";
```

Given the file is a standalone test script, the disable comment is cleaner.
  </action>
  <verify>npm run lint 2>&1 | grep "no-require-imports" | wc -l returns 0</verify>
  <done>ESLint test file passes without no-require-imports errors</done>
</task>
