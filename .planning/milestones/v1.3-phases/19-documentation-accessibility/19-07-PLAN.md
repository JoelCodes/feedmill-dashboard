<task type="auto">
  <name>Task 1: Make customer rows keyboard accessible</name>
  <files>src/app/customers/page.tsx</files>
  <action>
Lines 108-159 have a clickable div row without keyboard support:
- click-events-have-key-events: needs onKeyDown handler
- no-static-element-interactions: needs role="button" or use a button element

The best fix is to convert the div to a button or add proper keyboard handling with role.

Option 1 (recommended): Add role, tabIndex, and keyboard handler:

```typescript
<div
  key={customer.id}
  data-customer-id={customer.id}
  onClick={() => handleRowClick(customer.id)}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      handleRowClick(customer.id);
    }
  }}
  role="button"
  tabIndex={0}
  className="flex cursor-pointer items-center py-3 hover:bg-[var(--bg-page)]"
>
```

Option 2: Use a button element with flex styling (cleaner semantically).

Use Option 1 for minimal changes.

  <files>src/app/settings/page.tsx</files>
  <action>
Line 104 has a label not associated with any control:
```typescript
<label className="mb-2 block text-sm font-bold">Theme</label>
<ThemeToggle />
```

The ThemeToggle component is a radiogroup, so the label should either:
1. Use aria-labelledby on the ThemeToggle
2. Or wrap in a fieldset with legend
3. Or use a heading/description instead of label

Since ThemeToggle renders a radiogroup with its own button labels, the simplest fix is to change from `<label>` to a regular span or paragraph element, since this is a section heading not a form label:

```typescript
<div>
  <span className="mb-2 block text-sm font-bold">Theme</span>
  <ThemeToggle />
</div>
```

Or use a proper heading structure:

```typescript
<div>
  <h3 className="mb-2 text-sm font-bold">Theme</h3>
  <ThemeToggle />
</div>
```

Use span approach to minimize structural changes.
  </action>
