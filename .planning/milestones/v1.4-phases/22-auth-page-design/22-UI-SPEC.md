## Interaction Patterns

| Interaction | Behavior | Visual Feedback |
|-------------|----------|-----------------|
| Click user area | Toggle dropdown menu | Dropdown appears below trigger |
| Hover user area | Highlight trigger | Background: var(--bg-page) with opacity |
| Click Sign Out | Close dropdown, trigger sign out | Menu closes, redirect to sign-in |
| Click outside dropdown | Close dropdown | Menu closes |
| Keyboard (Enter/Space) | Activate focused element | Same as click |
| Keyboard (Escape) | Close dropdown | Menu closes |

**Accessibility:**
- Dropdown trigger has `aria-expanded` state
- Menu has `role="menu"` with proper aria labels
- Focus trap within open dropdown
- Sign Out button has descriptive aria-label

**Source:** Standard dropdown pattern from NotificationDropdown.tsx (lines 116-138), Clerk UserButton behavior, WCAG 2.1 AA requirements from Phase 19.
