    Workflow structure per D-11:

    name: Production Smoke Tests

    Trigger: repository_dispatch with types [vercel.deployment.ready]

    Job: smoke-tests
    - runs-on: ubuntu-latest
    - Conditional: only run if github.event.client_payload.deployment.environment == 'production'

    Steps:
    1. Checkout code: actions/checkout@v4
    2. Setup Node.js: actions/setup-node@v4 with node-version '20' and cache 'npm'
    3. Install dependencies: npm ci
    4. Install Playwright browsers: npx playwright install --with-deps chromium
    5. Run smoke tests:
       - env:
         - PLAYWRIGHT_BASE_URL: ${{ github.event.client_payload.deployment.url }}
         - CLERK_TEST_USER_EMAIL: ${{ secrets.CLERK_TEST_USER_EMAIL }}
         - CLERK_TEST_USER_PASSWORD: ${{ secrets.CLERK_TEST_USER_PASSWORD }}
       - run: npx playwright test --project=production-smoke
    6. Notify Vercel of check status:
       - if: always()
       - uses: vercel/repository-dispatch/actions/status@v1
       - with: name "Production Smoke Tests"

    Create .github and .github/workflows directories as needed.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Secrets | Repository secrets containing test credentials |
| GitHub Actions to Vercel | Webhook communication for deployment status |
| GitHub Actions to Production | Playwright tests executing against live deployment |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-05 | Information Disclosure | GitHub Secrets | mitigate | Store test credentials in GitHub Secrets only; never log or expose in workflow outputs; use ${{ secrets.* }} syntax |
| T-24-06 | Tampering | Workflow file | mitigate | Workflow file committed to repo; GitHub branch protection prevents unauthorized modifications |
| T-24-07 | Denial of Service | Smoke test timeout | mitigate | 30s timeout per test prevents hung deployments; retries: 0 ensures quick failure feedback |
| T-24-08 | Spoofing | repository_dispatch | accept | Vercel webhook includes deployment metadata; verify environment == 'production' in workflow condition |
</threat_model>
